深度学习基本优化算法

随机梯度下降（SGD）

如我们之前讨论过的，从数据生成分布独立随机抽取m个样本，计算他们的梯度均值，就可以梯度的无偏估计。随机梯度下降就是每次从训练集中拿包含m个样本的小批量来计算梯度，更新参数。
SGD的一个关键参数是学习率。由于SGD的梯度估计是有方差的，所以SGD不会像全局梯度下降一样收敛在局部最低点（如果可以的话），而是在局部最低点附近震荡。如果想尽量靠近局部最低点，可以随着迭代次数缩小学习率。记第k次迭代的学习率为$\epsilon_k$,在实践中，一般会线性衰减直至第$\tau$步：$$\epsilon_k = (1-\alpha)\epsilon_0 + \alpha \epsilon_\tau$$其中$\alpha = \frac{k}{\tau}$在$\tau$步迭代后，一般学习率不变。一般的$\tau$应为几百步，$\epsilon_\tau$应为$\epsilon_0$的1%。
这么一看主要问题是设置$\epsilon_0$。我们可以通过观察代价函数随迭代次数的变化曲线来判断学习率是否合适。一般来说，在有下降的趋势下，温和的震荡（由SGD梯度估计不准确，梯度方向曲率过大，还有一些随机的正则化策略如随机噪声，Dropout，引起的）是好的。但是过大的震荡，代价函数明显增加，说明学习率可能太大了。另一方面，代价函数下降缓慢，代价函数卡在一个值较高的区域，说明学习率可能太小了。

动量

动量算法的基本思想是将前面迭代计算得到的梯度，加到本次迭代的方向里作为参考（当然要乘以衰减系数）。具体的表示我们将本次迭代的更新方向记为动量$v$，动量由本次迭代的梯度乘以学习率加上上次迭代的动量乘以衰减系数：$$v\gets \alpha v - \epsilon\nabla_\theta(\frac{1}{m}\sum_{i=1}^{m} L(f(x^{(i)};\theta),y^{(i)}))$$ $$\theta \leftarrow \theta + v$$动量算法能很大程度上缓解Hessian矩阵病态的问题。如图所示：代价函数在梯度方向曲率很大。导致只使用梯度方向作为优化方向的话，代价函数会在“狭谷”间来回震荡，下降很慢。这种情况下，真正好的方向是指向谷底的方向。而每次计算的梯度，都对谷底方向有一个分量，所以可以看成动量在这个方向一直“加速”，而梯度指向“狭谷”两侧的分量则在动量中相互抵消了。所以动量的方向会逐渐纠正梯度方向，向有利的方向前进。

由此我们也可以想到，在梯度较小但方向几乎不变的平坦区域。动量会在梯度方向一直加速（最快能使步长加到原来的$\frac{1}{1-\alpha}$倍），帮助较快的通过平坦区域。在实践中，$\alpha$一般取0.5,0.9,0.99,和学习率一样$\alpha$也需要随时间不断调整。一般初始取较小的值然后增大。
动量算法还有一种变式：Nesterov动量，其更新方式为：$$v\gets \alpha v - \epsilon\nabla_\theta(\frac{1}{m}\sum_{i=1}^{m} L(f(x^{(i)};\theta + \alpha v),y^{(i)}))$$ $$\theta \leftarrow \theta + v$$Nesterov中，计算梯度前先在参数上施加动量，可以解释为往标准动量方法中添加了一个校正因子。

自适应学习率算法

在学习过程中，损失可能在某些参数方向很敏感，而不敏感于其他。如果我们统一得对所有参数使用同样的学习率，可能就忽略了某些参数在优化上的特殊性质。动量算法可以某种程度解决这个问题，但是代价是引入了超参数$\alpha$。
Delta-bar-delta算法提供了这样一个思路：如果某个参数在优化中更新的方向一直不变则应该增大其的学习率，而方向改变则应减小其的学习率。不过这种方法只能在全批量优化中使用。接下来说一下在SGD中也能使用的自适应学习率算法。

AdaGrad

AdaGrad的调节参数学习率的策略是：使参数学习率反比于历史梯度平方和的平方根。即：$$\alpha_j \sim \frac{1}{\sqrt{\sum_{i=1}^n (\frac{\partial J}{\partial \theta_j})_i^2}}$$这样具有较大梯度的参数的学习率会快速下降，而较小梯度的参数的学习率会较慢下降。
净效果是参数在方向较为平缓的区域会取得较大的进步。
AdaGrad算法的过程如下：

有全局学习率$\epsilon$,初始参数$\theta$，小常数$\delta$（为了数值计算的稳定）。
初始化$r=0$。
计算小批量梯度：$g \leftarrow \frac{1}{m} \nabla_\theta \sum_i L(f(x^{(i)};\theta),y^{(i)})$
累计平方梯度：$r = r + g\odot g$
计算更新：$\Delta \theta \leftarrow -\frac{\epsilon}{\delta + \sqrt{r}} \odot g$
应用更新：$\theta \leftarrow \theta + \Delta \theta$

RMSProp

AdaGrad算法的缺陷在于：在深度网格上使用时，学习率可能过早的变得太小。由于深度网格的优化是非凸的。一般优化算法会穿过多个结构到达一个局部的凸碗，并最终收敛在凸碗的碗底。在到达凸碗时，可以看成是一个局部的凸优化。此时之前的历史梯度对当前优化的学习率已经没有指导意义了，反而可能把学习率缩放的太小使之难以到达这个凸结构。
RMSProp算法在AdaGrad算法上进行了优化，使其在非凸优化中有一个更好的表现。它在累计平方梯度这一步更改为：$$r \leftarrow \rho r + (1-\rho)g \odot g$$RMSProp算法通过缩放因子$\rho$遗弃较早的历史梯度，使学习率不会过早的变得过小。在到达凸碗时，可以看作是在凸碗处初始化了一个AdaGrad算法,并较快收敛。

二阶近似方法
牛顿法

牛顿法通过一次将函数优化到其二阶近似的最优点来加速优化过程。代价函数在$\theta_0$的二阶近似为：$$J(\theta) \approx J(\theta_0) + g^T (\theta - \theta_0) + \frac{1}{2} (\theta - \theta_0)^T H (\theta - \theta_0)$$将上式对$\theta$求偏导并使之等于0有：$$g + H(\theta - \theta_0) = 0$$ $$\theta^* = \theta_0 - H^{-1}g$$牛顿法的收敛速度很快，但是有两大问题：
1.在参数为n的情况下，H是n*n的矩阵。在深度学习的情况下，参数尤其的多，H尤其的大。这样H的求逆代价很高。
2.牛顿法不保证H矩阵一定正定，这也意味着不保证$H^{-1}g$方向上代价函数一定是下降的。这个性质也会导致使用牛顿法容易卡在鞍点。

对于第二点有强制正则化策略：$$\theta^* = \theta_0 - (H + \alpha I)^{-1}g$$来保证是$H + \alpha I$是正定的。

共轭梯度法

对于一阶优化算法，如最速梯度法。其会在每一步迭代中达到该步优化方向的最优点。但是在下一步更新完成后，参数在上一步优化方向上已经不是最优点了。相当于前一步的优化效果在后一步中被浪费一很大一部分，这也造成了优化缓慢。
共轭梯度法在选取当前优化方向上，会综合之前每一次的更新方向，使之当前方向与之前每一次更新方向都共轭。这样，在更新当前步时，参数在之前方向上依然是最优。更新方向的形式如下：$$d_t = \nabla_\theta J(\theta) + \beta_t d_{t-1}$$一般$d_t^T H d_{t-1}$称为$d_t,d_{t-1}$共轭。但是这涉及到计算H的特征值，计算代价过高。
不过可以有替代计算方法：
（1）Fletcher-Reeves：$$\beta_t = \frac{\nabla_\theta J(\theta_t)^T \nabla_\theta J(\theta_t)}{\nabla_\theta J(\theta_{t-1})^T \nabla_\theta J(\theta_{t-1})}$$
(2) Polak-Ribiere:$$\beta_t = \frac{(\nabla_\theta J(\theta_t)-\nabla_\theta J(\theta_{t-1}))^T \nabla_\theta J(\theta_t)}{\nabla_\theta J(\theta_{t-1})^T \nabla_\theta J(\theta_{t-1})}$$

BFGS

BFGS方法是拟牛顿法的一种。拟牛顿法的思路在于用另一个n阶矩阵G把$H^{-1}$给替换掉。G满足拟牛顿条件且为正定矩阵。所谓拟牛顿条件，如我们在牛顿法中叙述的：在k此迭代后，代价函数二阶近似的梯度为:$$\nabla J(\theta) = g_k + H_k(\theta - \theta_k)$$令$\theta = \theta{k+1}$有：$$g_{k+1} - g_k = H_k(\theta_{k+1} - \theta_k)$$此为拟牛顿条件，$G_k$必须要满足。$G_0$要设为正定对称矩阵。具体的迭代公式就不放了，但是可以保证之后的每一个G都是正定的。
